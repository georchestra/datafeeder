# Internationalization (i18n)

## Configuration

The app uses ngx-translate for internationalization:

```typescript
import { provideI18n, TRANSLATE_DEFAULT_CONFIG } from "geonetwork-ui";

export const appConfig: ApplicationConfig = {
  providers: [
    provideI18n(TRANSLATE_DEFAULT_CONFIG),
    // ...
  ],
};
```

## Translation Files

Translation JSON files are in `translations/`:

```
translations/
├── en.json
├── fr.json
├── de.json
├── it.json
├── es.json
├── nl.json
├── pt.json
└── sk.json
```

## Using Translations in Templates

### TranslateDirective

Use the `translate` directive for static text:

```html
<h1 translate>import.title</h1>
<button translate>common.save</button>
```

### TranslatePipe

Use the pipe for dynamic translations:

```html
<p>{{ 'import.status.' + status | translate }}</p>
```

## Using Translations in TypeScript

Inject `TranslateService`:

```typescript
import { TranslateService } from "@ngx-translate/core";

export class MyComponent {
  private readonly translate = inject(TranslateService);

  getMessage() {
    return this.translate.instant("import.message");
  }

  async getAsyncMessage() {
    return await lastValueFrom(this.translate.get("import.message"));
  }
}
```

## Marking Strings for Extraction

Use the `marker` function to mark strings for extraction:

```typescript
import { marker } from "@biesbjerg/ngx-translate-extract-marker";

const messages = {
  success: marker("import.success"),
  error: marker("import.error"),
};
```

## Extracting Translations

Run the extraction command to update translation files:

```bash
npm run i18n:extract
```

This command:

1. Scans all source files
2. Extracts translation keys
3. Updates all JSON files
4. Formats the output

## Translation Key Structure

Use hierarchical keys with dots:

```json
{
  "import": {
    "title": "Import Data",
    "status": {
      "pending": "Pending",
      "success": "Success",
      "error": "Error"
    },
    "actions": {
      "submit": "Submit",
      "cancel": "Cancel"
    }
  },
  "common": {
    "save": "Save",
    "delete": "Delete",
    "edit": "Edit"
  }
}
```

## Best Practices

1. **Group by feature**: Use feature prefixes (`import.`, `metadata.`, `events.`)
2. **Reuse common strings**: Put shared strings in `common.`
3. **Extract immediately**: Run `i18n:extract` after adding new strings
4. **Use semantic keys**: `import.title` not `import.text1`
5. **Mark all user-visible text**: Never hardcode user-facing strings
