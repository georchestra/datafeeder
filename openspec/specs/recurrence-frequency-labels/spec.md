# Capability: Recurrence Frequency Labels

## Purpose

Defines how recurrence presets and custom cron schedules are displayed to users in the UI. Labels must be human-readable descriptions using i18n preset keys or cronstrue for custom crons, without exposing raw scheduling details.

## Requirements

### Requirement: Recurrence frequency labels must not expose scheduling details

The recurrence display component MUST show:
- For a known preset: the corresponding translated label via i18n keys (e.g. `recurrence.preset.EVERY_DAY`, `recurrence.preset.EVERY_WEEK`, `recurrence.preset.EVERY_MONTH`, etc.)
- For a custom cron: the human-readable description generated by the cronstrue library, localised to the application language
- For no recurrence: a translated placeholder via the i18n key `recurrence.none`

The system MUST NO LONGER dynamically generate labels from `<quantity><unit>` codes.

#### Scenario: Preset label in French locale

- **WHEN** the preset is `EVERY_DAY` and the locale is `fr`
- **THEN** the displayed label MUST be « Tous les jours »

#### Scenario: Preset label in English locale

- **WHEN** the preset is `EVERY_WEEK` and the locale is `en`
- **THEN** the displayed label MUST be "Every week"

#### Scenario: Custom cron label

- **WHEN** the recurrence is a custom cron `30 2 15 * *` and the locale is `fr`
- **THEN** the displayed label MUST be the cronstrue description in the active language

#### Scenario: No recurrence label

- **WHEN** no recurrence is configured
- **THEN** the displayed label MUST be the translated placeholder (e.g. "No recurrence")
