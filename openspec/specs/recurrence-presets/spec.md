# Capability: Recurrence Presets

## Purpose

Defines the fixed set of recurrence presets available for scheduling recurring data ingestion, the API endpoints for listing and querying them, and the read-only display on the dataset detail page.

## Requirements

### Requirement: Fixed recurrence presets

The system MUST define a fixed set of recurrence presets, each associating an enum identifier with a cron expression. The supported presets MUST be:

| Preset         | Cron expression (default `RECURRENCE_EXECUTION_HOUR=4`) |
| -------------- | ------------------------------------------------------- |
| `EVERY_MINUTE` | `* * * * *`                                             |
| `EVERY_HOUR`   | `0 * * * *`                                             |
| `EVERY_DAY`    | `0 4 * * *`                                             |
| `EVERY_WEEK`   | `0 4 * * 1`                                             |
| `EVERY_MONTH`  | `0 4 1 * *`                                             |
| `EVERY_YEAR`   | `0 4 1 1 *`                                             |

The `EVERY_DAY`, `EVERY_WEEK`, `EVERY_MONTH`, and `EVERY_YEAR` presets use the `RECURRENCE_EXECUTION_HOUR` configuration parameter (default: 4) for the execution hour. This set is hard-coded on the backend. Adding a new preset requires a source code change.

#### Scenario: Backend exposes the preset list via API

- **WHEN** the frontend calls `GET /ingestion/recurrence-presets`
- **THEN** the response MUST contain an array of objects `{ "id": "<PRESET_ID>", "cron": "<expression>" }` for each defined preset

#### Scenario: Preset to cron mapping

- **WHEN** a user submits the preset `EVERY_DAY` during the process step
- **THEN** the cron expression stored in `IntegrityLink.schedule` MUST be `0 4 * * *`

#### Scenario: Unknown preset rejected

- **WHEN** a user submits a preset `EVERY_TWO_WEEKS` that does not exist in the enum
- **THEN** the backend MUST reject the request with HTTP 422

---

### Requirement: preset_id inline on GET /ingestion/integrity-link/{id}

The `GET /ingestion/integrity-link/{id}` response MUST include a `preset_id` field (the preset identifier corresponding to the current `schedule`, or `null`). This field is NOT stored in the database â€” it is resolved by reverse-lookup in `CRON_PRESET_MAP` (inverse of `PRESET_CRON_MAP`) on each call.

There is no dedicated `GET /ingestion/integrity-link/{id}/recurrence` endpoint.

#### Scenario: IntegrityLink with a cron matching a known preset

- **WHEN** the frontend calls `GET /ingestion/integrity-link/{id}` for an IntegrityLink whose `schedule` is `0 4 * * *`
- **THEN** the response MUST contain `"preset_id": "EVERY_DAY"`

#### Scenario: IntegrityLink with a custom cron (modified by an admin)

- **WHEN** the frontend calls `GET /ingestion/integrity-link/{id}` for an IntegrityLink whose `schedule` is `30 2 15 * *` (not recognised in `CRON_PRESET_MAP`)
- **THEN** the response MUST contain `"preset_id": null`

#### Scenario: IntegrityLink with no recurrence

- **WHEN** the frontend calls `GET /ingestion/integrity-link/{id}` for an IntegrityLink whose `schedule` is `null`
- **THEN** the response MUST contain `"preset_id": null`

### Requirement: Read-only recurrence display on the events page

The UI MUST display the current recurrence of a dataset on the events page (`/:intlink_id/events`) in a disabled (read-only) combobox. The combobox MUST show:
- The translated preset label if the value matches a known preset
- A human-readable cron description via cronstrue if the value is a custom cron
- A translated placeholder (e.g. "No recurrence") if no schedule is configured

#### Scenario: Known preset displayed on the events page

- **WHEN** the user is on the events page of a dataset whose recurrence has `preset_id = "EVERY_DAY"` and `cron = "0 4 * * *"`
- **THEN** the combobox MUST display the corresponding translated label (e.g. "Every day" in English)
- **AND** the combobox MUST be disabled (not editable)

#### Scenario: Custom cron displayed on the events page

- **WHEN** the user is on the events page of a dataset whose recurrence has `preset_id = null` and `cron = "30 2 15 * *"`
- **THEN** the combobox MUST display the human-readable description generated by cronstrue
- **AND** the combobox MUST be disabled (not editable)

#### Scenario: No recurrence displayed on the events page

- **WHEN** the user is on the events page of a dataset whose recurrence has `preset_id = null` and `cron = null`
- **THEN** the combobox MUST display a translated placeholder (e.g. "No recurrence")
- **AND** the combobox MUST be disabled (not editable)

#### Scenario: cronstrue localisation

- **WHEN** the application locale is `fr`
- **THEN** the cronstrue description MUST be displayed in French
