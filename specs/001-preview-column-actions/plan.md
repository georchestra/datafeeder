# Implementation Plan: Actions sur les colonnes de l'aperçu tabulaire

**Branch**: `001-preview-column-actions` | **Date**: 2026-02-18 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-preview-column-actions/spec.md`

## Summary

Add column-level actions (rename, remove, change type, filter) to the tabular preview in the data ingestion wizard. Users configure transformations directly from column headers. The transformation configuration is persisted incrementally via PUT metadata and applied during both preview (GET preview with `raw` fallback) and final ingestion (process DAG). An explicit Pydantic model replaces the untyped `integrity_transformation` dict.

## Technical Context

**Language/Version**: Python 3.12 (backend, data_manipulation), TypeScript/Angular 20 (frontend)
**Primary Dependencies**: FastAPI, SQLModel, Pydantic v2, pandas, geopandas (backend); Angular Material, Tailwind CSS, geonetwork-ui 2.9.0 (frontend)
**Storage**: PostgreSQL (datakern schema for IntegrityLink, staging schema for data tables)
**Testing**: pytest (Python), vitest (frontend unit tests)
**Target Platform**: Linux server (Docker containers)
**Project Type**: Web application (monorepo: backend + frontend + ELT + shared library)
**Performance Goals**: Preview response < 3 seconds (SC-002)
**Constraints**: Preview limited to 10 rows max (FR-017a). Debounce 400ms for text inputs.
**Scale/Scope**: Single wizard flow, ~4 new frontend components, ~3 backend model changes, ~2 data_manipulation module changes
